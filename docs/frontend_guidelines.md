# pWPS Agent 前端体验与多端适配指南 frontend_guidelines.md

## 1. 文档定位

本文档约束 pWPS Agent 前端工作台的用户体验、信息呈现、多端适配和交互设计原则。

本系统不是普通聊天页面，而是面向焊接工艺草案生成的字段确认工作台。

---

## 2. 总原则

1. 用户友好优先于功能堆叠。
2. 字段确认卡片优先于聊天框。
3. 关键风险必须显性展示。
4. 证据默认摘要展示，支持逐层展开。
5. 移动端、平板端、桌面端都必须可用。
6. 前端不实现 pWPS 推理逻辑。
7. 用户任何一步都应知道：当前确认什么、为什么确认、系统建议是什么、风险在哪里、下一步是什么。

---

## 3. 页面结构

```text
任务创建页
字段确认工作台
证据查看面板
风险与审计面板
最终草案预览页
历史任务页
设置页
```

---

## 4. 字段确认工作台

核心 UI：字段确认卡片。

卡片包含：

```text
当前字段组名称
当前进度
已知条件摘要
系统推荐值
候选值列表
候选理由
证据摘要
风险标签
操作按钮
```

操作：

```text
接受推荐
选择备选
自定义输入
要求重新检索
跳过 / 标记待确认
查看证据
查看风险说明
```

---

## 5. 多端适配

### 桌面端

推荐三栏：

```text
左：字段进度 / 字段组导航
中：字段确认卡片
右：证据 / 风险 / 审计面板
```

### 平板端

推荐双栏：

```text
上/左：确认卡片
下/右：证据与风险 Tab
```

### 移动端

推荐单栏折叠：

```text
确认卡片
  -> 推荐值
  -> 候选值
  -> 风险
  -> 证据折叠
  -> 操作按钮 sticky bottom
```

---

## 6. 风险呈现

风险不能只出现在长文本中。必须有明确标签：

```text
低风险
中风险
高风险
硬冲突
待人工确认
证据不足
禁止推断
```

---

## 7. 前端禁止事项

1. 禁止前端自行推断字段值。
2. 禁止前端自行计算 publishability。
3. 禁止前端绕过后端 schema。
4. 禁止把证据隐藏在不可发现的位置。
5. 禁止移动端核心流程不可用。
6. 禁止为了炫技引入复杂动画影响可读性。
